Anarchy
Warzone
Battle Royale Vehicles Inventory System Multiplayer

A full-scale battle royale — last player standing, shrinking zone, looting, and a fleet of driveable vehicles spread across the map. Every vehicle type — bikes, cars, armoured trucks — had distinct physics, handling models, and combat interactions, all replicated live across multiplayer clients.

I was responsible for three of the hardest systems in any battle royale: vehicle physics from scratch, a full multiplayer inventory system, and making all of it work at scale across concurrent online sessions. No middleware — written in Unreal C++ end to end.

// My responsibilities
  • Built all vehicle types from the ground up — custom wheel physics, suspension simulation, torque curves, collision response, and networked movement for bikes, cars, and armoured vehicles.
  • Implemented a full multiplayer inventory system — item pickup, storage, equip/drop, and UI — all replicated with server authority, conflict resolution, and anti-cheat validation.
  • Architected client-server state replication for vehicle occupancy, weapon state, and player inventory with delta-compression to keep bandwidth under control at scale.
  • Owned cross-platform build pipeline — single codebase delivering consistent gameplay across different target devices.
  • Built and iterated the multiplayer UI/UX — health, inventory HUD, vehicle interaction prompts — all driven by replicated game state.
// System architecture
Vehicles
Custom Vehicle Physics
Built on UE's Chaos Vehicle plugin, extended with custom suspension and torque models per vehicle class. Networked via compressed transform replication — smooth interpolation on clients, authoritative simulation on server.
Inventory
Replicated Inventory
TArray-backed inventory component on each PlayerState, replicated with FFastArraySerializer for delta efficiency. Server validates all pickup/drop RPCs before committing state. Supports stacking, equip slots, and loot spawner integration.
Networking
BR Session Management
Online Subsystem session flow: lobby → map load → in-game → results. Zone shrink logic runs server-side, broadcast via multicast. Player elimination handled with deferred authority to prevent desync on kill confirmation.
