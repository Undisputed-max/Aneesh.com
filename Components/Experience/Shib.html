Eternity
Metaverse Live Multiplayer Cross-platform Team Lead iOS · Android · Windows

A Fortnite-scale persistent metaverse — one live world, multiple game modes running simultaneously, players moving between experiences without session reloads. Think social hub plus competitive mini-games, all connected across three device classes in real time.

Built at GamaSome. Led a team of 4. I owned the cross-platform networking layer, the multiplayer architecture, and the gameplay systems holding the entire platform together — from initial design through to shipping on mobile, iOS, and Windows.

// My role — Team Lead & Lead Programmer
  • Designed the full dedicated server architecture — session management, player replication, persistent world state across concurrent clients.
  • Built a platform abstraction layer — single Unreal codebase deploying to Android, iOS, and Windows with dynamic quality scaling per device budget.
  • Implemented a data-driven game mode router — players enter and exit mini-game instances inside the persistent world without a full session handoff.
  • Architected a modular avatar pipeline for digital human characters — parameter-driven animation mapping, API-reactive facial/body states, extensible without new assets per variation.
  • Led sprint planning, code reviews, and cross-discipline coordination with art and design teams.
// System architecture
Networking
UE5 Dedicated Server
Authoritative server model with client-side movement prediction and rollback. Custom replication channels to minimise bandwidth for mobile clients. Seamless session joins without full map reload.
Cross-platform
Platform Abstraction
Input, rendering budget, and texture streaming all abstracted per platform at runtime. Scalability groups defined per device class. Single CI/CD pipeline outputs Android APK, iOS IPA, and Windows build simultaneously.
Gameplay
Game Mode Router
Data-asset-driven game mode registry. World partition used to stream mini-game zones in/out. Player state preserved across mode transitions via a custom save-component, no reconnect required.
Avatar
Digital Human Pipeline
Control rig + blend space driven by runtime parameters rather than hardcoded states. API callbacks update animation variables directly, transitioning characters from static assets to live-reactive digital humans.
Performance
Rendering Optimisation
Nanite disabled on mobile paths; instanced static meshes and LOD budgets tuned per platform. Shader complexity reduced for mobile GPU constraints. Frame time profiled with Unreal Insights.
Team
4-Person Lead
Planned and tracked sprints in Jira. Defined coding standards and review gates. Resolved daily blockers between art, rigging, and engineering. Took game from vertical slice to cross-platform release.
