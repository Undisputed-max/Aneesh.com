Card Battle Clash Royale-style Solo Dev Mobile Full Stack — Design to Ship

A card-based real-time strategy game inspired by Clash Royale — players build a deck, spend elixir to deploy units onto the battlefield, and race to destroy the opponent's three towers before theirs fall. Fast, readable, and mobile-first.

Every line of code, every system, every piece of logic — built by me alone over 1.5 years. Card deployment, unit AI, tower health, elixir economy, collision-driven combat, matchmaking — all of it from scratch in Unreal C++. This is the project that taught me how games actually hold together end to end.

// What I built — solo, from scratch
  • Card system and deck builder — data-asset-driven card definitions (unit type, stats, cost), deck validation, and in-match hand/elixir management loop.
  • Unit AI and pathfinding — behaviour tree-driven unit logic: march toward target tower, prioritise nearest threat, attack on range trigger, death and reward events.
  • Tower damage and win condition — three-tower destruction system with networked health replication and game-end evaluation logic.
  • Elixir economy system — server-authoritative elixir tick, card cost deduction with client-side prediction and server reconciliation.
  • Collision-driven combat — deterministic hit resolution using physics overlap events, damage type dispatch, and knockback response.
  • Mobile UI and UX — full match HUD, card hand display, drag-to-deploy input, and end-of-match result screen.
// System architecture
Card System
Data-Asset Card Pipeline
Each card is a UDataAsset defining unit class, mesh, stats, and elixir cost. Deck stored as TArray on PlayerState. Hand manager draws from deck and cycles on card use. All card deployment is server-authoritative.
AI
Behaviour Tree Units
UE Behaviour Tree per unit type. Blackboard keys: target actor, attack range, health threshold. EQS used to select optimal march path toward enemy towers. Units share a perception component to react to nearby enemies mid-march.
Combat
Collision-driven Resolution
Attack radius modelled as sphere overlap, triggers damage event on server. Damage types defined as UDamageType subclasses. Tower health component broadcasts OnDestroyed delegate — game mode listens and evaluates win condition.
